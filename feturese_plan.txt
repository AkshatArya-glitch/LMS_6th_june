EEPL LMS WEBSITE + ADMIN CMS FEATURE PLAN

Goal:
Build the website so the admin can update almost every frontend section without editing code. The public website should become API-driven, and the admin panel should work like a simple CMS for home page content, courses, webinars, trusted-by logos, testimonials, blogs, articles, certificates, settings, media, SEO, and LMS learning content.


1. Current Project Review

Current frontend:

- Static HTML/CSS/JavaScript website.
- Existing pages include home, courses, webinar, articles, certification, contact, and user dashboard.
- Most visible content is currently stored in JavaScript files such as courses-data.js, webinar-data.js, articles-data.js, certification-data.js, and dashboard mock data.
- Images and logos are stored inside Frontend/assets.

Current backend:

- PHP/MySQL API with JWT authentication.
- Existing modules include students, admins, courses, modules, lessons, enrollments, progress, quizzes, answers, quiz attempts, and certificates.
- Backend has a schema.sql file with core LMS tables.
- Backend needs content management tables and APIs for website sections.

Main requirement now:

- Admin should be able to update the website from an admin dashboard.
- No need to open code files for normal content changes.
- Public website should fetch content from backend APIs.
- Admin should manage homepage sections, course pages, webinars, trusted logos, testimonials, blogs, articles, certificates, contact info, menus, footer, images, SEO, and general settings.

9. YouTube Video Storage For Course Lessons

Requirement:

- Admin can upload or attach course videos.
- Videos can be stored in YouTube and shown inside lessons.

Important note:

- YouTube does not provide a true private folder feature through API.
- Use YouTube playlists as course folders, or keep grouping inside LMS database.
- Private YouTube videos are difficult for many students because each viewer may need Google account permission.
- For LMS access, unlisted videos plus enrollment checks are easier.
- For stronger security, use Vimeo, Bunny Stream, Cloudflare Stream, or signed HLS storage.

Required Google/YouTube APIs:

- YouTube Data API v3
- OAuth 2.0 Client ID
- OAuth consent screen
- Scopes:
  - https://www.googleapis.com/auth/youtube.upload
  - https://www.googleapis.com/auth/youtube
  - https://www.googleapis.com/auth/youtube.readonly

Admin APIs:

- POST /v1/admin/media/youtube/upload
- GET /v1/admin/media/youtube/status/{id}
- POST /v1/admin/courses/{courseId}/youtube-playlist
- POST /v1/admin/lessons/{lessonId}/video


10. Google Meet Live Classes

Requirement:

- Admin can schedule live classes.
- Students can see upcoming classes and join from dashboard.
- Google Meet links can be created or stored.

Recommended approach:

- Use Google Calendar API to create events with Google Meet conference links.
- Store the returned Meet URL in the LMS.
- Use Google Meet API only if advanced meeting-space or conference-record features are needed.

Required Google APIs:

- Google Calendar API
- Optional Google Meet REST API
- OAuth 2.0 Client ID
- OAuth consent screen
- Scopes:

14. Security And Admin Rules

- Only authenticated admins can access admin APIs.
- Use role permissions: super_admin, admin, teacher, content_manager.
- Keep API keys and Google tokens only in backend.
- Never expose secrets in frontend JavaScript.
- Validate image/file uploads.
- Restrict upload file types and file size.
- Add slug uniqueness checks.
- Add status field for draft/published content.
- Keep created_by and updated_by fields for admin activity.
- Add soft delete for important content.
- Add audit log for critical changes.
- Use HTTPS in production.
